⚓ Naval Warfare RTS - 배포 가이드

📌 게임 크기 비율 (mapandunit.png 기준)
- 건물 (본부, 조선소 등): 100 x 100 (기준 크기)
- 자원 구역: 반지름 60 (건물의 60%)
- 유닛 (군함): 30 x 30 (건물의 30%)

이 비율은 mapandunit.png의 초록색 사각형(건물), 빨간색 선(자원), 파란색 선(유닛)의 비율을 반영합니다.

🌐 배포 방법

1. Render.com (추천 - 무료, 쉬움)

장점: 무료, 데이터베이스 지속, Socket.io 완벽 지원

  1. Render.com 가입
  2. New + → Web Service 선택
  3. GitHub 저장소 연결 또는 수동 배포
  4. 설정: 
    Name: naval-warfare-rts
Environment: Node
Build Command: npm install
Start Command: npm start
Instance Type: Free
  5. Environment Variables 설정: 
    JWT_SECRET=your-random-secret-key-here-change-this
  6. Create Web Service 클릭

배포 완료 후 제공되는 URL로 접속하세요!

2. Railway.app (무료, 빠름)

장점: 배포가 매우 빠름, GitHub 연동 자동

  1. Railway.app 가입
  2. New Project → Deploy from GitHub repo
  3. 저장소 선택 또는 업로드
  4. 자동으로 감지되어 배포됨
  5. Variables에서 환경 변수 설정: 
    JWT_SECRET=your-random-secret-key
  6. Settings → Generate Domain으로 URL 생성

3. Heroku (유명함)

장점: 안정적, 많은 사용자

  1. Heroku 가입 (신용카드 필요)
  2. Heroku CLI 설치: 
    npm install -g heroku
  3. 명령어 실행: 
    heroku login
heroku create naval-warfare-rts
git init
git add .
git commit -m "Initial commit"
git push heroku main
  4. 환경 변수 설정: 
    heroku config:set JWT_SECRET=your-secret-key

4. Glitch.com (가장 쉬움)

장점: 웹에서 바로 편집, GitHub 없이 배포

  1. Glitch.com 가입
  2. New Project → Import from GitHub 또는 파일 업로드
  3. .env 파일 생성: 
    JWT_SECRET=your-secret-key
PORT=3000
  4. 자동으로 배포됨
  5. Share 버튼으로 URL 확인
⚠️ 배포 전 체크리스트
  • ✅ JWT_SECRET을 강력한 랜덤 문자열로 변경
  • ✅ game.db는 git에 올리지 말 것 (.gitignore 확인)
  • ✅ 서버 로그에서 개인정보 제거
  • ✅ 포트는 process.env.PORT로 자동 설정됨

🆘 문제 해결

  • 게임창이 안 뜸: 브라우저 콘솔(F12)에서 에러 확인
  • 연결 안 됨: 서버가 실행 중인지, 포트가 열려있는지 확인
  • 데이터 날아감: SQLite 데이터베이스(game.db) 백업 필요
  • 속도 느림: 무료 플랜은 제한이 있음, 업그레이드 고려
